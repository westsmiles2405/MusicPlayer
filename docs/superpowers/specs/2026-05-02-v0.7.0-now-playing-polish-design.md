# MusicPlayer v0.7.0 Now Playing 大屏与 Apple Music 体验强化 · 设计文档

- 创建日期：2026-05-02
- 版本：v0.7.0
- 文档状态：Draft v1.0
- 基于：v0.6.0（搜索 / 收藏 / 最近播放已完成）

---

## 1. 背景

v0.6.0 已把“找得到内容并能回访”补齐：搜索、收藏、最近播放均已接入现有队列和资料库路径。当前阶段最主要缺口不再是功能数量，而是播放体验完成度：播放器缺少沉浸式主播放界面、系统级快捷操作尚未闭环、主题与文案存在分散实现，且“Apple Music 风格”尚未被写成硬验收标准。

因此 v0.7.0 定位为**Apple Music 体验强化版本**：在不扩展功能面、不改动核心数据模型的前提下，完成 Now Playing 大屏、连续转场、视觉材质层、快捷键冲突矩阵、主题跟随系统、i18n 收敛。

## 2. 目标 / 非目标

### 2.1 目标

1. 新增 Now Playing 大屏覆盖层，与现有 MiniPlayer 和播放状态体系联动。
2. 建立统一快捷键行为：空格播放/暂停、`⌘F` 跳转并聚焦搜索，并定义清晰冲突优先级。
3. 建立统一主题跟随系统：前端自动跟随 macOS/Web 系统深浅色。
4. 补齐 v0.6/v0.7 页面与组件文案的 i18n key，消除新增路径上的硬编码文案。
5. 将“Apple Music 风格”转成可验收规范：大封面主视觉、封面取色背景、玻璃控制区、MiniPlayer 到大屏连续转场。
6. 保持 v0.6 已有导航与页面行为稳定，不引入破坏性改动。

### 2.2 非目标

- 不做全站视觉重构（布局体系、组件库、设计语言大改）。
- 不新增歌词、均衡器、可视化、智能推荐或新的资料库功能。
- 不改动 Rust 音频引擎核心架构，不让 PCM 经过 IPC。
- 不引入手动主题切换开关（v0.7 仅做系统跟随）。

## 3. 版本定位

v0.7.0 是：

**Now Playing 大屏 + 连续转场 + 视觉一致性**

它的目标是把现有能力“做完整、做顺手”，而不是“再加一批新页面”。这与 v0.5.0（资料库/播放列表）和 v0.6.0（搜索/收藏/最近播放）的节奏保持一致：每个版本聚焦一个可验收主题。

## 4. 信息架构与交互入口

### 4.1 入口规则

1. MiniPlayer 主区域点击：打开 Now Playing 大屏。
2. 大屏关闭方式固定为：顶部返回/下拉按钮 + `Esc`。点击背景关闭不作为必做能力。
3. 不新增一级导航路由入口；Now Playing 以覆盖层形态工作，不改变主路由信息架构。

### 4.2 路由策略

v0.7.0 固定采用“状态驱动覆盖层”，而不是单独页面路由。原因：

- 不影响当前 `/search`、`/library/*`、`/playlists/*` 的深链接稳定性。
- 关闭大屏后可无缝回到原上下文。
- 与 MiniPlayer 的语义一致（同一播放器，不是新业务页面）。

## 5. 模块设计

### 5.1 前端模块

新增或扩展模块职责如下：

1. `NowPlayingOverlay`：展示 + 交互容器，读取 `usePlayer()` 状态并调用现有播放控制动作。
2. `useKeyboardShortcuts`：统一注册/清理快捷键监听；将按键映射到页面导航与播放器动作，并处理冲突场景。
3. `useSystemTheme`：监听系统主题变化，向全局状态同步。
4. `NowPlayingTransition`（使用 Framer Motion 承载）：负责 MiniPlayer -> Now Playing 的共享元素转场。
5. i18n 文案层：新增 v0.7 key 并补齐 v0.6 漏网 key，页面仅消费 key。

### 5.2 现有模块边界保持

- `src/repositories/` 仍是唯一 IPC 边界。
- 页面/组件不直接调用 Tauri `invoke()`。
- 播放命令继续通过既有 `usePlayer()` -> repository -> Tauri command 路径下发。

## 6. Now Playing 设计

### 6.1 UI 组成

Now Playing 大屏至少包含：

1. 大封面主视觉（封面位于核心视觉区域，非角落小图）。
2. 当前曲目信息（标题、艺人、专辑）。
3. 进度与时长控制（复用现有 seek 能力）。
4. 核心播放控制（上一首、播放/暂停、下一首），播放按钮为大圆形主按钮。
5. 音量与静音入口（可复用现有组件逻辑）。
6. 背景层（封面取色 + 模糊/渐变）与玻璃控制区。

交互反馈要求：

1. 进度条、音量条、主要按钮必须具备 `hover` / `active` / `disabled` 状态。
2. 深色模式下文本与控件需保持可读对比度。

### 6.2 数据来源与状态

- 数据只读来源：`usePlayer()` 暴露的当前播放状态。
- 不新增“第二套播放器状态”。
- 对 `missingAt` 曲目的行为保持既有规则：不可播放项不触发播放动作。

### 6.3 转场设计

MiniPlayer 与 Now Playing 必须体现“同一个播放器展开”，不允许无动画硬切：

1. MiniPlayer 点击后，封面元素从 MiniPlayer 位置过渡到大屏封面区域。
2. 曲目信息使用淡入过渡，不做瞬时替换。
3. 背景模糊层在 `150-250ms` 内渐显。
4. 关闭时执行对称反向过渡。
5. 若共享布局动画成本过高，最低要求为 `opacity + scale + blur` 组合过渡。

### 6.4 Apple Music 风格背景层

背景分三层：

1. `CoverColorBackground`：从当前封面提取主/辅色，生成径向渐变。
2. `BlurredCoverLayer`：封面图放大后模糊，透明度按主题调整。
3. `GlassControlPanel`：控制区为半透明玻璃容器。

平台策略：

1. Web / WebView 使用 `backdrop-filter`。
2. macOS 原生材质接入不在 v0.7 阶段强制要求直接落地 `NSVisualEffectView`，但保留后续桥接位。

性能约束：

1. 不做多层全屏实时 blur 叠加。
2. 背景取色结果应缓存，避免每帧重复取色。
3. 动画期间不重复执行取色。
4. 低性能环境允许退化为静态渐变 + 弱模糊。

### 6.5 性能与降级

- 大屏动画与背景效果优先保证稳定帧率。
- Web 端允许弱化背景效果，但不允许破坏布局与核心控制能力。

## 7. 快捷键与系统媒体键设计

### 7.1 键位定义（v0.7 固定）

1. `Space`：播放 / 暂停。
2. `⌘F`：桌面端跳转到 `/search` 并聚焦搜索输入框。

### 7.2 冲突与优先级

快捷键忽略场景固定如下：

1. `input` / `textarea` / `contenteditable` 聚焦时，`Space` 不触发全局播放切换。
2. slider 拖动中，`Space` 不触发全局播放切换。
3. dialog/modal 打开时，`Esc` 先关闭最上层 dialog，再关闭 Now Playing。
4. Web Demo 可保留浏览器默认 `⌘F` 查找行为，不强制拦截。

### 7.3 系统媒体键边界

媒体键不通过前端 `useKeyboardShortcuts` 实现，统一由 Rust `system::now_playing` 路径负责：

1. 若 v0.4.0 媒体键桥接已完成，v0.7 只做回归验证和补漏。
2. 若主干存在缺口，媒体键可用性为 v0.7 阻塞项。

## 8. 主题跟随系统

### 8.1 行为定义

- 应用主题自动跟随系统（深色/浅色）。
- 系统主题切换时，前端样式状态实时同步。
- v0.7 不提供手动开关，避免状态来源双轨并存。

### 8.2 实现约束

- 主题状态应有单一来源（store 或 hook），由布局层统一下发。
- 不允许页面各自监听系统主题并自行改 class，避免闪烁与竞态。

## 9. i18n 收敛

### 9.1 范围

1. v0.6 新页面（搜索/收藏/最近播放）补齐未收敛文案。
2. v0.7 新增 Now Playing 与快捷键提示文案全部走 i18n key。

### 9.2 规则

- 页面/组件不出现新增硬编码中文。
- i18n key 命名按现有命名空间风格延续，不引入第二套命名规范。
- v0.7 仍以中文资源为主，英文资源可保留占位但需结构完整。

## 10. 错误处理

1. 快捷键触发失败（例如播放器状态异常）应走现有错误通道，不吞错。
2. Now Playing 打开时若无当前曲目，展示明确空状态而非空白页。
3. 主题监听异常时回退到默认主题并记录错误日志。

## 11. 测试策略

### 11.1 前端

至少覆盖：

1. Now Playing 打开/关闭交互与核心控制按钮行为。
2. MiniPlayer -> Now Playing 过渡存在性（不允许无动画硬切）。
3. `Space` 与 `⌘F` 快捷键映射和输入框/slider/dialog 冲突场景。
4. 背景层与玻璃控制区在深浅色模式下可读。
5. 系统主题变化后 UI 状态同步。
6. v0.7 相关页面 i18n key 可解析且无新增硬编码。

### 11.2 Rust / 后端

- 媒体键桥接走回归测试优先策略，确保 `system::now_playing` 相关命令不回归。
- 无新增数据库模型时，不引入不必要迁移测试。

## 12. 验收标准

v0.7.0 完成时必须满足：

1. 可从 MiniPlayer 打开并关闭 Now Playing 大屏。
2. MiniPlayer -> Now Playing 存在连续过渡动画，不允许无动画硬切。
3. Now Playing 包含大封面、标题、艺人、专辑、进度、上一首、播放/暂停、下一首、音量/静音。
4. 背景基于当前封面生成取色渐变 + 模糊层；无封面时使用稳定默认渐变。
5. 控制区具备半透明玻璃视觉，深色/浅色下可读。
6. 大屏播放控制与进度 seek 与 MiniPlayer 状态完全一致，不新增第二套播放器状态。
7. `Space` 与 `⌘F` 全局行为符合定义，并正确处理输入框、slider、dialog 冲突。
8. 系统媒体键行为与 Rust `system::now_playing` 路径一致，不出现前端/后端双轨冲突。
9. 系统主题切换时应用外观同步更新，无明显闪烁。
10. v0.6/v0.7 新增文案已收敛到 i18n key。
11. 不破坏 v0.6 的搜索、收藏、最近播放主流程。
12. 低性能或 Web Demo 环境允许弱化背景模糊，但不能破坏基本布局和控制能力。

## 13. 与后续版本边界

v0.8.0+ 再考虑：

- 更激进的视觉重构（布局体系和动效体系升级）
- 歌词、可视化、均衡器等扩展能力
- 手动主题策略或多主题系统

这样可以保证 v0.7.0 聚焦“体验闭环完善”，避免范围膨胀导致交付失焦。
