# MusicPlayer v0.5.0 Library Playlists Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the v0.5.0 daily-use local library UI: sidebar navigation, recent/songs/albums/artists pages, album/artist detail pages, playlist CRUD, playlist detail, missing-track handling, and queue playback from every list.

**Architecture:** Keep Rust as the source of truth for library and playlist data, with all IPC isolated in `src/repositories/`. Add a real React `AppShell` with nested routes, reusable library table/grid components, and React Query cache invalidation around scan and playlist mutations. Tighten playlist backend contracts first so UI drag-and-drop uses stable 0-based `playlistPosition` values.

**Tech Stack:** Tauri 2.x, Rust, SQLite/rusqlite, React 19, React Router 7, React Query 5, Zustand, Vitest, Testing Library, Tailwind CSS 4, lucide-react.

## Conventions

- Start implementation from an isolated worktree:

```bash
git worktree add .worktrees/v0.5.0-library-playlists -b feat/v0.5.0-library-playlists
cd .worktrees/v0.5.0-library-playlists
```

- Do not call `invoke()` outside `src/repositories/`.
- Do not send PCM over IPC.
- Use `CARGO_TARGET_DIR=/tmp/musicplayer-target` for Rust commands.
- Keep playlist positions 0-based and contiguous after every remove/reorder.
- v0.5.0 preserves the current schema behavior that permits the same `track_id` to appear multiple times in one playlist at different positions. Position compaction and reorder must therefore identify rows by SQLite `rowid`, not by `(playlist_id, track_id, position)` alone.
- Library lists hide missing tracks; playlist details keep missing tracks visible, disabled for playback, removable.
- Every task ends with a commit. If the local environment blocks git index writes, leave the file changes staged instructions in the final task note.

- [ ] **Step 3: Implement `PlaylistTrackView` and compact positions**

In `src-tauri/src/db/playlists.rs`, add:

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct PlaylistTrackView {
    #[serde(flatten)]
    pub track: crate::db::tracks::TrackView,
    pub playlist_position: i64,
}

fn validate_name(name: &str) -> AppResult<&str> {
    let trimmed = name.trim();
    if trimmed.is_empty() {
        return Err(crate::error::AppError::InvalidInput(
            "playlist name cannot be empty".into(),
        ));
    }
    Ok(trimmed)
}

fn compact_positions(conn: &Connection, playlist_id: i64) -> AppResult<()> {
    let mut stmt = conn.prepare(
        "SELECT rowid, position FROM playlist_tracks
         WHERE playlist_id = ?1
         ORDER BY position, added_at, rowid",
    )?;
    let rows: Vec<(i64, i64)> = stmt
        .query_map(params![playlist_id], |row| Ok((row.get(0)?, row.get(1)?)))?
        .collect::<Result<_, _>>()?;
    drop(stmt);

    for (new_pos, (rowid, _old_pos)) in rows.into_iter().enumerate() {
        conn.execute(
            "UPDATE playlist_tracks
             SET position = ?2
             WHERE rowid = ?1",
            params![rowid, new_pos as i64],
        )?;
    }
    Ok(())
}
```

Update `create` and `rename` to call `validate_name(name)?` and store the trimmed value. Change `get_tracks` to return `Vec<PlaylistTrackView>` and select `pt.position AS playlist_position`. Change `remove_track` to call `compact_positions(conn, playlist_id)?` after a successful delete.

Replace `reorder` with a transaction that:

1. Calls `compact_positions(conn, playlist_id)?` before opening the transaction.
2. Reads `COUNT(*)`.
3. Rejects `source_position < 0`, `destination_position < 0`, `source_position >= count`, or `destination_position >= count` with `AppError::InvalidInput`.
4. Uses `conn.unchecked_transaction()` because the current repository helper signature is `reorder(conn: &Connection, ...)`; do not switch this API to `&mut Connection` unless `Database::with_conn` is changed in the same task.
5. Selects the moved row's SQLite `rowid` by `playlist_id + source_position`.
6. Moves that `rowid` to position `-1`, shifts rows, writes the destination position by `rowid`, commits, then calls `compact_positions(conn, playlist_id)?` again.

If `conn.unchecked_transaction()` causes a borrow conflict in the final code, use a manual transaction with `conn.execute_batch("BEGIN IMMEDIATE")`, explicit `COMMIT`, and explicit `ROLLBACK` on error. Do not leave reorder as multiple unguarded statements.

const gradients = [
  "linear-gradient(135deg, #f04d5f, #235a7a)",
  "linear-gradient(135deg, #d85a2a, #2d7c6f)",
  "linear-gradient(135deg, #7a4cc2, #c94f63)",
  "linear-gradient(135deg, #2f6f9f, #d0a642)",
];

function pickGradient(seed: string) {
  const total = Array.from(seed).reduce((sum, ch) => sum + ch.charCodeAt(0), 0);
  return gradients[total % gradients.length];
}

export function CoverArt({
  coverPath,
  title,
  seed,
  size = "md",
}: {
  coverPath: string | null;
  title: string;
  seed?: string | number;
  size?: "sm" | "md" | "lg";
}) {
  if (coverPath) {
    return (
      <img
        className={`cover-art cover-art--${size}`}
        src={coverPath}
        alt={title}
      />
    );
  }

  return (
    <div
      className={`cover-art cover-art--${size} cover-art--placeholder`}
      aria-label={title}
      role="img"
      style={{ background: pickGradient(String(seed ?? title)) }}
    />
  );
}
